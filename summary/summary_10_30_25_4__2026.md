## What was added

### `system-backend/infrastructure/saml-proxy-shibboleth/`

- **Docker Compose** (`docker-compose.yml`): LDAP (389 DS from Unicon testbed) + **Shibboleth IdP** image built from `idp/Dockerfile` (`FROM unicon/shibboleth-idp:3.4.3`).
- **IdP overlay** (`idp/shibboleth-idp/`): metadata for Nest SP, static **UAM IdP** metadata (`uam-idp-metadata.xml` from [AMU SSO](https://sso.amu.edu.pl/simplesaml/saml2/idp/metadata.php)), local `idp-metadata.xml`, `metadata-providers.xml`, `relying-party.xml`, and `attribute-filter.xml` updates for the Nest SP entity ID.
- **Jetty** (`idp/shib-jetty-base/start.d/*.ini`): PKCS#12 paths point at `/opt/shibboleth-idp/credentials/*.p12` instead of Docker secrets.
- **`idp.properties`**: entityID `https://localhost:14443/idp/shibboleth`, scope `local.test`, signing/encryption keys and sealer JCEKS on the filesystem.
- **Dev certificates** (`certs/` and `idp/shibboleth-idp/credentials/`): locally generated RSA keys, SP cert for Nest, IdP signing/encryption (same key for dev), Jetty `idp-browser.p12` / `idp-backchannel.p12` (password `changeme`), `sealer.jks`, LDAP TLS placeholder cert, **`uam-idp-signing.pem`** (UAM signing leaf).
- **`openssl-min.cnf`**: avoids broken `OPENSSL_CONF` on some Windows installs when regenerating certs.
- **`README.md`**: architecture, run instructions, honest note that **live IdP→UAM SAML delegation** still needs institutional Shibboleth + UAM configuration beyond this repo.
- **`scripts/verify-saml-proxy-metadata.ps1`**: optional smoke checks for Nest SP metadata and proxy IdP TLS.

## Operational notes

- Proxy SSO URL for Nest: `https://localhost:14443/idp/profile/SAML2/Redirect/SSO` (mapped host port **14443** → container **4443**).
- Demo LDAP logins: **`staff1` / `password`**, **`student1` / `password`**.
- Nest must use **`SAML_IDP_CERT`** (or `_PATH`) for the **proxy IdP** signing cert when using the proxy, not UAM’s cert.
